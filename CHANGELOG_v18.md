# TextChecker v18 - Відновлення режиму обробки та покращення UI чорного списку

## Зміни в версії 18

### 1. Відновлення секції "Режим обробки"

У версії v17 секція "Режими обробки" була повністю видалена. У версії v18 вона відновлена, але з **тільки одним чекбоксом**:

**Відновлено:**
- Секція "2. Режим обробки"
- Чекбокс "Тільки очищення HTML (без перевірки правил перекладу)"
- За замовчуванням чекбокс **вимкнено** (користувач може вибрати режим)

**НЕ відновлено (видалено назавжди):**
- Чекбокс "Пропускати рядки, де _checked вже заповнено" - більше не потрібен

**Оновлена нумерація секцій:**
1. Вибір файлу та стовпчиків
2. Режим обробки ← відновлено
3. Керування та словники ← змінено з "2." на "3."

### 2. Поле "Дія" в Чорному списку - тепер радіокнопки замість чекбоксу

У версії v17 поле "дія" було представлено як чекбокс з текстом "Активно", що було незрозуміло для користувачів.

**Раніше (v17):**
- ☑ **Активно** (галочка ✓) = термін буде автоматично видалятися
- ☐ **Неактивно** (порожнє) = термін тільки підсвічується

**Тепер (v18):**
- 🔘 **Видаляти** - автоматично видаляє речення з забороненим терміном
- 🔘 **Повідомляти** - тільки позначає в звіті для ручної перевірки

Це робить інтерфейс значно зрозумілішим - користувач бачить чітко дві опції та вибирає потрібну.

### Технічні зміни

#### У файлі `text_checker.py`:

1. **Змінено дефолтне значення `html_only_var`:**
   ```python
   self.html_only_var = tk.BooleanVar(value=False)  # За замовчуванням вимкнено
   ```

2. **Відновлено секцію `options_frame`:**
   ```python
   options_frame = ttk.LabelFrame(self.root, text="2. Режим обробки")
   options_frame.pack(fill='x', padx=10, pady=5)

   self.chk_html_only = ttk.Checkbutton(
       options_frame,
       text="Тільки очищення HTML (без перевірки правил перекладу)",
       variable=self.html_only_var
   )
   self.chk_html_only.pack(side='left', padx=5)
   ```

3. **Замінено чекбокс на радіокнопки в `RowEditDialog`:**
   ```python
   if col == 'дія' and dictionary_key == 'blacklist':
       var = tk.StringVar(value="Видалити")
       
       radio_frame = ttk.Frame(main_frame)
       radio_frame.grid(row=idx, column=1, sticky='w', pady=5, padx=5)
       
       radio1 = ttk.Radiobutton(radio_frame, text="Видаляти", variable=var, value="Видалити")
       radio1.pack(side='left', padx=(0, 15))
       
       radio2 = ttk.Radiobutton(radio_frame, text="Повідомляти", variable=var, value="Повідомляти")
       radio2.pack(side='left')
   ```

4. **Оновлено логіку збереження для підтримки `StringVar`:**
   ```python
   if isinstance(widget, tk.StringVar):
       # Для радіокнопок зберігаємо значення напряму
       self.result[col] = widget.get()
   ```

5. **Оновлено логіку завантаження значень:**
   - "Видалити", "Видаляти", "✓" або будь-яке непорожнє значення → встановлюється "Видалити"
   - "Повідомляти", "Підсвітити" або порожнє → встановлюється "Повідомляти"

**README.md** оновлено:
