# TextChecker (v14)

Інструмент для контент-менеджерів, який очищує HTML-описи, формує `*_checked` колонки для RU/UA та підсвічує проблемні місця в Excel-файлах.

## Ключові можливості

- Автовизначення RU/UA колонок (з пріоритетом `Описание;1` та `Описание (ua);1`)
- Дві колонки результату: `RU_checked` і `UA_checked` (за фактично обраними назвами)
- Окрема колонка **`Статус`** з значеннями на кшталт:
  - `OK`
  - `Потрібен переклад`
  - `Заборонений термін`
  - `Технічний HTML`
  - `Китайський текст`
  - `Підозріло короткий опис`
  - `Пропущено (вже оброблено)`
- Збереження початкового порядку секцій при форматуванні HTML
- Консервативне видалення дубльованих пунктів списку `<li>` (збереження порядку)
- Автоочищення порожніх абзаців (`<p> </p>`, порожніх `<div>` тощо)
- Нормалізація типових символів і чисел (наприклад, `57,5` → `57.5`, `1,87` → `1.87`)
- Пропуск вже оброблених рядків (якщо `_checked` вже заповнено)
- Пакетна обробка: один файл, кілька файлів або вся папка
- Режим **"Тільки очищення HTML"** (без перевірки правил перекладу)
- Кнопка **"Відкрити результат"** після завершення
- Підсвітка:
  - **червона** — мовна невідповідність (потрібен переклад),
  - **жовта** — інші помилки/ризики.

## Використання

1. Запустіть застосунок:
   ```bash
   python "python error search v.6.py"
   ```
2. Оберіть:
   - один/кілька Excel-файлів, або
   - папку з `.xlsx` файлами.
3. Перевірте/підтвердьте RU та UA колонки.
4. Оберіть режими:
   - **Пропускати рядки, де `_checked` вже заповнено** (увімкнено за замовчуванням),
   - **Тільки очищення HTML** (опційно).
5. Натисніть **"Почати перевірку"**.
6. Після завершення перегляньте зведений звіт та відкрийте результат кнопкою **"Відкрити результат"**.

## Що додається в результат

Для кожного обробленого файлу створюється файл з суфіксом `*_checked.xlsx`, де додаються/оновлюються:

- `Помилки перекладу`
- `Статус`
- `<RU-колонка>_checked`
- `<UA-колонка>_checked`

## Словники

- `translation_rules.txt` — правила виявлення хибних перекладів RU→UA
- `ignore_rules.txt` — фрази-винятки
- `blacklist_terms.txt` — заборонені/підозрілі терміни

Формат:

```txt
# translation_rules.txt
російський_корінь = український_корінь1, український_корінь2
```

```txt
# ignore_rules.txt / blacklist_terms.txt
кожен термін або фраза з нового рядка
```
