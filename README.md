# TextChecker (v14)

Інструмент для контент-менеджерів, який очищує HTML-описи, формує `*_checked` колонки для RU/UA та підсвічує проблемні рядки у файлах Excel.

## Ключові можливості

- Автовизначення RU/UA колонок (з пріоритетом `Описание;1` та `Описание (ua);1`)
- Дві колонки результату: `RU_checked` і `UA_checked` (за фактично обраними назвами)
- Окрема колонка **`Статус`** з значеннями на кшталт:
  - `OK`
  - `Потрібен переклад`
  - `Заборонений термін`
  - `Технічний HTML`
  - `Китайський текст`
  - `Підозріло короткий опис`
  - `Пропущено (вже оброблено)`
- Збереження початкового порядку секцій при форматуванні HTML
- Консервативне видалення дубльованих пунктів списку `<li>` (збереження порядку)
- Автоочищення порожніх абзаців (`<p> </p>`, порожніх `<div>` тощо)
- Нормалізація типових символів і чисел (наприклад, `57,5` → `57.5`, `1,87` → `1.87`)
- Пропуск вже оброблених рядків (якщо `_checked` вже заповнено)
- Пакетна обробка: один файл, кілька файлів або вся папка
- Режим **"Тільки очищення HTML"** (без перевірки правил перекладу)
- Кнопка **"Відкрити результат"** після завершення
- Підсвітка:
  - **червона** — мовна невідповідність (потрібен переклад),
  - **жовта** — інші помилки/ризики.

## Використання

1. Запустіть застосунок:
   ```bash
   python text_checker.py
   ```
2. Оберіть:
   - один/кілька Excel-файлів, або
   - папку з `.xlsx` файлами.
3. Перевірте/підтвердьте RU та UA колонки.
4. Оберіть режими:
   - **Пропускати рядки, де `_checked` вже заповнено** (увімкнено за замовчуванням),
   - **Тільки очищення HTML** (опційно).
5. Натисніть **"Почати перевірку"**.
6. Після завершення перегляньте зведений звіт та відкрийте результат кнопкою **"Відкрити результат"**.

## Що додається в результат

Для кожного обробленого файлу створюється файл з суфіксом `*_checked.xlsx`, де додаються/оновлюються:

- `Помилки`
- `Статус`
- `<RU-колонка>_checked`
- `<UA-колонка>_checked`

## Словники

Для зручного управління всіма словниками використовуйте кнопку **"📚 Словники"** в інтерфейсі програми. Відкриється єдине вікно з табличним інтерфейсом для кожного словника.

### Можливості табличного менеджера словників:
- **Вкладки**: `🔄 Помилки перекладу`, `✓ Винятки`, `⛔ Чорний список`
- **Таблична форма**: Перегляд та редагування даних у зручному табличному форматі
- **Додати/Редагувати/Видалити**: Управління записами через кнопки або подвійний клік
- **Пошук**: Фільтрація записів у поточному словнику
- **Статистика**: Перегляд кількості записів у кожному словнику
- **Експорт в Excel**: Експорт словника у файл Excel для зовнішнього редагування
- **Зберегти всі**: Збереження змін в усіх словниках одночасно
- **Перезавантажити**: Повернення до збережених версій файлів

### Формат словників (CSV):

Програма використовує формат CSV, який можна редагувати як в програмі, так і в Excel/Calc:

#### `translation_rules.csv`
Колонки:
- **російський_корінь** — корінь російського слова для пошуку
- **українські_корені** — неправильні українські варіанти (через кому)
- **виключення** — фрази-винятки, коли правило не застосовується (через кому)
- **тип** — категорія помилки (наприклад, "Помилка перекладу", "Суржик")
- **коментар** — пояснення чому це помилка

Приклад:
```csv
російський_корінь,українські_корені,виключення,тип,коментар
питани,харчуванн,"збалансоване харчування,дитяче харчування",Помилка перекладу,"Питание (електричне) → живлення, а не харчування"
```

#### `ignore_rules.csv`
Колонки:
- **фраза** — фраза-виняток (українською)
- **категорія** — категорія винятку (опційно)
- **коментар** — пояснення (опційно)

Приклад:
```csv
фраза,категорія,коментар
збалансоване харчування,Харчування,Допустимий контекст для слова "харчування"
```

#### `blacklist_terms.csv`
Колонки:
- **термін** — заборонений термін або фраза
- **категорія** — тип заборони (наприклад, "URL", "Контакти")
- **коментар** — пояснення (опційно)

Приклад:
```csv
термін,категорія,коментар
www.,URL,Заборонені веб-адреси в описах
```

### Міграція з TXT на CSV

При першому запуску програма автоматично конвертує старі `.txt` файли в новий `.csv` формат:
- `translation_rules.txt` → `translation_rules.csv`
- `ignore_rules.txt` → `ignore_rules.csv`
- `blacklist_terms.txt` → `blacklist_terms.csv`

Після міграції обидва формати зберігаються для сумісності. Програма використовує CSV файли, якщо вони доступні.

### Файли словників (застарілий TXT формат):

Для сумісності зі старими версіями:

- `translation_rules.txt` — правила виявлення хибних перекладів RU→UA
- `ignore_rules.txt` — фрази-винятки
- `blacklist_terms.txt` — заборонені/підозрілі терміни

Формат:

```txt
# translation_rules.txt
російський_корінь = український_корінь1, український_корінь2
```

```txt
# ignore_rules.txt / blacklist_terms.txt
кожен термін або фраза з нового рядка
```
