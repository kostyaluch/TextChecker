# Підсумок змін в TextChecker v17

## Виконані завдання

### 1. ✅ Підтримка копіювання/вставки з кириличної розкладки

**Що додано:**
- Ctrl+С (кирилична С) - копіювати
- Ctrl+В (кирилична В) - вставити  
- Ctrl+М (кирилична М) - вставити (альтернатива)
- Ctrl+Ч (кирилична Ч) - вирізати

**Де працює:**
- Головне вікно програми (SpellCheckerApp)
- Менеджер словників (DictionaryManager)
- Табличний менеджер словників (TableDictionaryManager)
- Діалог редагування записів (RowEditDialog)

**Реалізація:**
Додано методи `bind_cyrillic_shortcuts()`, `handle_copy()`, `handle_paste()`, `handle_cut()` до кожного вікна. Ці методи перехоплюють кириличні клавіші та генерують стандартні події Tkinter `<<Copy>>`, `<<Paste>>`, `<<Cut>>`.

### 2. ✅ Видалення розділу "Винятки" зі Словників

**Що змінено:**
- Видалено вкладку '✓ Винятки' з DictionaryManager
- Видалено вкладку '✓ Винятки' з TableDictionaryManager
- Оновлено статистику для відображення тільки 2 словників

**Залишились вкладки:**
- 🔄 Помилки перекладу (translation_rules.csv)
- ⛔ Чорний список (blacklist_terms.csv)

**Примітка:** Файли ignore_rules.csv/txt зберігаються для зворотної сумісності, але не відображаються в UI. Функціональність винятків перенесена в колонку "виключення" у translation_rules.csv (це було зроблено в v16).

### 3. ✅ Поле "Дія" в Чорному списку - тепер чекбокс

**Раніше:**
Текстове поле, в яке потрібно було вводити "Видалити" або "Підсвітити"

**Тепер:**
Чекбокс з підписом "Активно":
- ✓ (галочка) = термін буде автоматично видалятися
- (порожнє) = термін тільки підсвічується

**Реалізація:**
- Оновлено `RowEditDialog.__init__()` для визначення типу словника через параметр `dictionary_key`
- Додано логіку створення чекбоксу замість текстового поля для колонки "дія" в blacklist
- Оновлено метод `save()` для збереження "✓" або "" в залежності від стану чекбоксу
- Оновлено виклики `RowEditDialog` в `add_row()` та `edit_row()` для передачі `dictionary_key`

### 4. ✅ Режим "Тільки очищення" - єдиний режим роботи

**Що змінено:**
- Видалено секцію "2. Режими обробки" з UI
- Видалено чекбокс "Пропускати рядки, де _checked вже заповнено"
- Видалено чекбокс "Тільки очищення HTML"
- Перенумеровано секцію "3. Керування та словники" → "2. Керування та словники"

**Логіка:**
- `self.html_only_var` тепер завжди `True` (режим очищення завжди увімкнений)
- `self.skip_processed_var` встановлено в `False` (не використовується)

**Результат:**
Програма завжди працює в режимі "Тільки очищення HTML" без перевірки зі словником правил перекладу.
