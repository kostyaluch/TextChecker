# Changelog v16

## Основні зміни

### 1. Міграція винятків з ignore_rules.csv в translation_rules.csv

Всі винятки з окремого файлу `ignore_rules.csv` були автоматично співставлені та перенесені в колонку "виключення" відповідних правил у `translation_rules.csv`.

**Результати міграції:**
- Всього правил: 25
- Правил з винятками: 6
- Винятки автоматично співставлено на основі українських коренів

**Приклад:**
```csv
російський_корінь,українські_корені,виключення,тип,коментар
питани,харчуванн,"збалансоване харчування,дитяче харчування,здорове харчування",Помилка контексту,...
```

### 2. Заповнення типів у translation_rules.csv

Для кожного правила автоматично визначено тип помилки на основі коментарів та контексту.

**Розподіл за типами:**
- Помилка перекладу: 13 правил
- Суржик: 7 правил
- Помилка контексту: 5 правил

**Типи помилок:**
1. **Помилка перекладу** — коли перекладач обирає неправильне значення слова
2. **Суржик** — калькування з російської мови
3. **Помилка контексту** — коли перекладач плутає контекст використання слова

### 3. Розширення blacklist_terms.csv

Додано дві нові колонки:

#### Колонка "виключення"
Фрази (через кому), в контексті яких термін є допустимим. Якщо знайдено виняток, термін ігнорується.

**Приклад:**
```csv
термін,виключення,дія,категорія,коментар
аналог,оригінальний аналог,Підсвітити,Підробки,Допустимий в контексті "оригінальний аналог"
```

#### Колонка "дія"
Визначає, як програма обробляє знайдений термін:

- **Видалити** — автоматично видаляє речення або пункт списку (як раніше працювало)
- **Підсвітити** — тільки позначає рядок у звіті та вказує, яке слово знайдено (для ручної перевірки контент-менеджером)

**Різниця в повідомленнях:**
- Видалити: `[ЗАБОРОНЕНИЙ ТЕРМІН] Знайдено 'www.' у тексті: "..." (речення буде видалено)`
- Підсвітити: `[ПІДОЗРІЛИЙ ТЕРМІН] Знайдено 'samsung' у тексті: "..." (потребує перевірки)`

### 4. Оновлена логіка обробки чорного списку

#### Перевірка винятків
Перед позначенням терміну як забороненого, програма перевіряє чи немає винятків у всьому тексті рядка. Якщо знайдено виняток — термін ігнорується.

#### Дві дії для термінів
- Терміни з дією "Видалити" працюють як раніше — речення/пункти списку видаляються автоматично
- Терміни з дією "Підсвітити" — речення залишаються, але рядок позначається в звіті для ручної перевірки

### 5. Оновлений інтерфейс

**TableDictionaryManager** автоматично підхоплює нові колонки:
- У вкладці "🔄 Помилки перекладу" тепер видно колонку "виключення" та "тип"
- У вкладці "⛔ Чорний список" додано колонки "виключення" та "дія"

## Технічні зміни

### Оновлені константи
```python
DEFAULT_BLACKLIST_CSV_HEADER = ['термін', 'виключення', 'дія', 'категорія', 'коментар']
APP_VERSION = "v16"
```

### Оновлені функції

#### `parse_blacklist()`
Повертає список словників замість простого списку:
```python
[
    {
        'term': 'www.',
        'exceptions': [],
        'action': 'Видалити'
    },
    ...
]
```

#### `sentence_contains_blacklist(sentence, blacklist)`
Повертає словник з інформацією про знайдений термін або `None`:
```python
{
    'found': True,
    'term': 'www.',
    'action': 'Видалити'
}
```

#### `format_description(text, blacklist)`
Тепер враховує дію терміну:
- Пропускає абзаци тільки якщо `action == 'Видалити'`
- Якщо `action == 'Підсвітити'`, абзац залишається

### Міграція з TXT

При міграції blacklist з `.txt` в `.csv` формат:
- Автоматично додається колонка "виключення" (пуста)
- Автоматично встановлюється дія "Видалити" (за замовчуванням, як було раніше)

## Зворотна сумісність

- Програма підтримує обидва формати (CSV та TXT) для всіх словників
- Якщо CSV файл відсутній, використовується TXT (legacy режим)
- Старі словники автоматично мігруються при першому запуску

## Що далі?

Файл `ignore_rules.csv` зберігається для сумісності, але його використання більше не потрібне, оскільки всі винятки тепер в `translation_rules.csv`. Ви можете:
1. Залишити його як є (програма продовжить його підтримувати)
2. Видалити його, якщо не потрібна зворотна сумісність зі старими версіями

```csv

аналог,оригінальний аналог,Підсвітити,Підробки,Перевірити контекст
```

- `Видалити` → речення буде видалено автоматично
- `Підсвітити` → речення залишиться, але рядок буде позначено в звіті
