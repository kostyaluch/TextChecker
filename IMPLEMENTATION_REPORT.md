**Реалізовано:**

#### 3.1. Новий Файл
- **`rules_catalog.txt`** - єдиний каталог правил з 4 типами:

  - `IGNORE` - винятки (правильні фрази)
  - `AUTO_REMOVE` - автоматичне видалення заборонених термінів
  - `FLAG_ONLY` - позначення в помилках без видалення

## Інструкції для Локальної Перевірки

1. **Запустити GUI:**
   ```bash
   python "python error search v.6.py"
   ```

2. **Перевірити міграцію:**
   - При першому запуску `rules_catalog.txt` має автоматично створитися з legacy файлів
   - Перевірити лог-повідомлення: "Створено rules_catalog.txt з legacy-файлів"

3. **Перевірити нову кнопку:**
   - Натиснути "📝 Словник Правил"
   - Має відкритися редактор з вмістом `rules_catalog.txt`

4. **Перевірити обробку:**
   - Вибрати тестовий XLSX файл
   - Вибрати колонки RU та UA
   - Натиснути "▶ Почати перевірку"
   - Перевірити, що створюється `*_checked.xlsx`
   - Переконатися, що:
     - AUTO_REMOVE терміни видалені з форматованого тексту
     - FLAG_ONLY терміни присутні в колонці "Помилки"
     - ERROR_REPLACEMENT працює як раніше
     - IGNORE правильно фільтрує помилки

5. **Перевірити сумісність:**
   - Legacy кнопки все ще працюють
   - Можна редагувати окремі файли (перехідний період)
