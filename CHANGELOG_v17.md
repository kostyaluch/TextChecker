# TextChecker v17 - Покращення UI та UX

## Зміни в версії 17

### 1. Підтримка кириличної розкладки клавіатури

Додано підтримку копіювання/вставки з кириличної розкладки клавіатури:

- **Ctrl+С** (кирилична С) = копіювати
- **Ctrl+В** (кирилична В) = вставити
- **Ctrl+М** (кирилична М) = вставити (альтернатива)
- **Ctrl+Ч** (кирилична Ч) = вирізати

Підтримка працює в:
- Головному вікні програми
- Менеджері словників (DictionaryManager)
- Табличному менеджері словників (TableDictionaryManager)
- Діалогах редагування записів (RowEditDialog)

### 2. Видалення вкладки "Винятки" зі словників

Розділ "Винятки" видалено з обох менеджерів словників (текстового та табличного), оскільки функціональність винятків перенесена в колонку "виключення" словника "Помилки перекладу" у версії 16.

Залишились тільки дві вкладки:
- 🔄 **Помилки перекладу** (translation_rules.csv)
- ⛔ **Чорний список** (blacklist_terms.csv)

Примітка: файли `ignore_rules.csv` та `ignore_rules.txt` зберігаються для сумісності зі старими версіями, але більше не відображаються в UI.

### 3. Поле "Дія" в Чорному списку - тепер чекбокс

У табличному редакторі Чорного списку поле "дія" змінено з текстового поля на чекбокс:

**Раніше:**
- Потрібно було вводити текст "Видалити" або "Підсвітити"

**Тепер:**
- ☑ **Активно** (галочка ✓) = термін буде автоматично видалятися
- ☐ **Неактивно** (порожнє) = термін тільки підсвічується

Це робить інтерфейс інтуїтивнішим та зменшує ймовірність помилок введення.

### 4. Спрощення режимів обробки

Видалено секцію "2. Режими обробки" з інтерфейсу:
- Видалено чекбокс "Пропускати рядки, де _checked вже заповнено"
- Видалено чекбокс "Тільки очищення HTML (без перевірки правил перекладу)"

Програма тепер **завжди працює в режимі "Тільки очищення HTML"** - це найбільш затребуваний режим роботи.

Змінено нумерацію секцій:
- ~~3.~~ **2. Керування та словники**

### Технічні зміни

- Додано методи `bind_cyrillic_shortcuts()`, `handle_copy()`, `handle_paste()`, `handle_cut()` до всіх діалогових вікон
- Оновлено `RowEditDialog` для підтримки чекбоксу для поля "дія" при `dictionary_key='blacklist'`
- Видалено вкладку 'ignores' з `DictionaryManager.dictionaries` та `TableDictionaryManager.dictionaries`
- Оновлено методи `update_statistics()` для роботи тільки з двома словниками
- Змінено дефолтні значення: `self.html_only_var = tk.BooleanVar(value=True)`
